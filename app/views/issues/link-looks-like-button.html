Background

We have a button style (class="button") that is used to make inputs of type button, buttons and links look like buttons.

When a link looks like a button a user with voice recognition software (such as Dragon Naturally Speaking) will/may have problems using the link as they will call it a button.

There are many instances on GOV.UK of a role="button" being added to such links as there is a expectation that voice recognition software will then treat is as a button. Is this assumption correct?

Buttons should be activated with the space bar. When you add a role="button" to a link you need to add some javascript so that the link/button can be activated with the space bar. This is not happening on GOV.UK.
